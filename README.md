# Global CO2 Emissions — Data Visualization Project

A portfolio data-visualization project exploring **who drives global CO2
emissions, how that has changed since 1850, and what factors explain the
differences between countries** — built with pandas, matplotlib, seaborn,
networkx, and geopandas, using the object-oriented (stateless) matplotlib
interface throughout.

## Research question

How have global CO2 emissions evolved over time, who emits the most today
(in total and per person), and which factors — wealth, energy use,
population, fuel mix — are most associated with the differences across
countries?

## Data source

**Our World in Data — CO2 and Greenhouse Gas Emissions Dataset**
https://github.com/owid/co2-data (CC-BY 4.0)

- `data/owid-co2-data.csv` — the raw dataset, 50,000+ rows, 229
  countries/regions, 1750–2024, 79 variables (CO2 by fuel source, GDP,
  population, energy use, methane, nitrous oxide, temperature
  attribution, and more).
- `data/countries.geo.json` — world country boundary polygons (ISO-3
  coded) from the `johan/world.geo.json` GitHub repo, used to draw the
  choropleth map without depending on any paid/rate-limited mapping
  service.

`clean_data.py` derives two clean files from the raw CSV (regenerated
automatically every time `main.py` runs):

- `data/co2_countries_clean.csv` — country-level rows only (regional and
  income-group aggregates like "World", "Asia", "High-income countries"
  removed), trimmed to the columns actually used by the visualizations.
- `data/co2_world_clean.csv` — the "World" aggregate rows only, used for
  the global trend and fuel-mix charts.

## Setup (VS Code / terminal)

1. Put every `.py` file and `requirements.txt` in one folder.
2. Create a `data/` subfolder inside it and place `owid-co2-data.csv`
   and `countries.geo.json` inside `data/`.
3. (Recommended) create and activate a virtual environment:
   ```
   python -m venv venv
   venv\Scripts\activate        # Windows
   source venv/bin/activate     # macOS/Linux
   ```
4. Install dependencies:
   ```
   pip install -r requirements.txt
   ```
5. Run the whole pipeline:
   ```
   python main.py
   ```

`main.py` runs `clean_data.py` first, then each `viz*.py` script in
order, and prints a pass/fail summary. All 7 PNGs appear in `output/`.
You can also run any single `vizN_*.py` file on its own — each one is
fully self-contained and just needs `data/` to exist alongside it.

## The 7 visualizations

| # | File | Type | What it shows |
|---|------|------|----------------|
| 1 | `viz1_trajectory.py` | **Line chart** (Matplotlib) | Global CO2 emissions, 1850–2024, with era annotations |
| 2 | `viz2_top_emitters.py` | **Bar chart** (Matplotlib) | Top 10 emitters by total CO2 vs. top 10 by CO2 per capita — two different rankings |
| 3 | `viz3_gdp_scatter.py` | **Scatter/bubble chart** (Matplotlib) | GDP per capita vs. CO2 per capita; bubble size = population, color = energy use per capita; points have no outlines and are semi-transparent so overlaps stay readable |
| 4 | `viz4_source_mix.py` | **Stacked area chart** (Matplotlib) | Global CO2 emissions broken down by fuel source (coal/oil/gas/cement), 1900–2023 |
| 5 | `viz5_world_map.py` | **Choropleth map** (geopandas + Matplotlib) | CO2 emissions per capita by country, 2023 |
| 6 | `viz6_correlation.py` | **Heatmap** (seaborn + Matplotlib) | Spearman correlations between emissions, GDP, energy use, population, methane, N2O, and temperature attribution |
| 7 | `viz7_network.py` | **Network graph** (networkx + Matplotlib) | Top-30 emitting countries linked when their coal/oil/gas/cement emissions "fingerprint" is highly similar; node size = total CO2, node color = CO2 per capita |

## Design notes

- All plots use the **object-oriented (stateless) matplotlib interface**
  (`fig, ax = plt.subplots()`, then `ax.set_xlabel(...)`,
  `fig.savefig(...)`, etc.) rather than the stateful `plt.xlabel()` /
  `plt.savefig()` "current axes" style.
- Plain **white backgrounds, no gridlines** on every chart.
- Legends/colorbars are either removed when unnecessary or kept deliberately
  informative — e.g. the scatter plot's colorbar encodes a genuinely
  different variable (energy use per capita) than the axes, instead of
  repeating information already shown on the y-axis.
- The scatter plot uses semi-transparent, outline-free points
  (`alpha=0.6`, `edgecolors="none"`) so overlapping bubbles stay readable
  instead of stacking into a solid dark mass.
